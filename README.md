# SynthHub

SynthHub -- one unified API for differentially private synthetic data.

```python
from synthhub import Synthesizer

synth = Synthesizer(method="independent", epsilon=1.0, random_state=0)
synth.fit(real_df)
synthetic_df = synth.sample(1000)
report = synth.evaluate(real_df, synthetic_df, target="label")
```

## Same Dataset, Same Epsilon Benchmark

The MVP includes the benchmark harness before publishing claimed results. Run
`examples/compare_backends.ipynb` after installing optional backends to refresh
this table on the same dataframe with `epsilon=1.0`.

| Method | Backend | Epsilon spent | Utility score | Re-ID risk | Status |
|---|---|---:|---:|---:|---|
| AIM | Private-PGM | 1.0 | pending | pending | adapter registered |
| MST | Private-PGM | 1.0 | pending | pending | adapter registered |
| PrivBayes | DataSynthesizer | 1.0 | generated by notebook | generated by notebook | optional |
| MWEM/PAC-Synth | SmartNoise | 1.0 | generated by notebook | generated by notebook | optional |
| DP-CTGAN/PATE-CTGAN/PATE-GAN/DP-GAN/QUAIL | SmartNoise | 1.0 | generated by notebook | generated by notebook | optional, heavy |
| PrivBayes/PATEGAN/DPGAN | SynthCity | 1.0 | generated by notebook | generated by notebook | optional, Python-version sensitive |
| independent | SynthHub baseline | 1.0 | generated by notebook | generated by notebook | runnable |

Do not treat the `independent` baseline as a replacement for AIM, MST, or
PrivBayes. It exists so the API, evaluation, and privacy-report contracts are
executable in a clean install.

## Install

```bash
python -m pip install synthhub
```

For local development:

```bash
python -m pip install -e ".[test]"
python -m pytest -q
```

For AIM/MST experiments, use Python 3.10-3.12, install the optional Private-PGM
dependency, and make the Private-PGM `mechanisms` folder importable:

```bash
python -m pip install -e ".[private-pgm,test]"
```

On newer Python versions, the core SynthHub package still installs, but the
Private-PGM extra is skipped and the AIM/MST adapters raise a clear
`BackendNotAvailableError` until a compatible external engine is available.

Optional backend families:

```bash
python -m pip install "synthhub[smartnoise]"
python -m pip install "synthhub[synthcity]"
python -m pip install "synthhub[datasynthesizer]"
```

## API

```python
from synthhub import Synthesizer

synth = Synthesizer(method="aim", epsilon=1.0, delta=1e-9)
synth.fit(df)
sample = synth.sample(5000)
report = synth.evaluate(df, sample, target="outcome")

print(synth.privacy_report_.to_dict())
print(report.to_dict())
```

Supported method names:

| Method | Purpose |
|---|---|
| `aim` | Private-PGM AIM adapter; intended default DP marginal backend |
| `mst` | Private-PGM MST adapter |
| `privbayes`, `datasynthesizer-privbayes` | DataSynthesizer correlated-mode PrivBayes adapter |
| `datasynthesizer-independent` | DataSynthesizer independent-attribute DP adapter |
| `independent` | built-in one-way marginal smoke-test baseline |
| `mwem`, `pacsynth`, `dpctgan`, `patectgan`, `pategan`, `dpgan`, `quail` | SmartNoise Synthesizers adapters |
| `smartnoise-aim`, `smartnoise-mst` | SmartNoise AIM/MST adapters |
| `synthcity-privbayes`, `synthcity-pategan`, `synthcity-dpgan` | SynthCity privacy plugin adapters |

## Privacy Notes

SynthHub does not invent new DP algorithms. It wraps existing engines behind a
single dataframe API and records backend-reported privacy accounting.

Formal DP guarantees are conditional on public preprocessing metadata. If
SynthHub infers column types, categories, or numeric bounds from the private
dataframe, the fitted privacy report includes a warning. For formal releases,
pass an explicit public `Schema`.
